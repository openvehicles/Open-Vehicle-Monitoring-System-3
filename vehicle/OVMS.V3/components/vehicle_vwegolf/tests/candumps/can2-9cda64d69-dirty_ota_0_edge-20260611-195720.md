## Notes

First unfiltered FCAN capture (filter disabled via `xvg fcanfilter off`,
firmware 9cda64d69-dirty). 95 distinct IDs, ~1611 fps aggregate over 123 s.

### Filter-off storm test result

Module survived ~1700 fps unfiltered FCAN: no TWDT, no wdgreset, `rxovr=0`
(RXB0OVR heartbeat only — frames spill to RXB1, none lost at MCP level).
Soft stress visible: poller RX task queue overflows, duktape ticker.1 took
2.2 s, logger dropped 1822/200185 (0.9%). Verdict: filter-off OK for short
supervised captures only; keep filter on otherwise.

### Timeline (rel. to first frame)

- t=0–9.1: EVSE handshake window. `BMS_PowerBus.BatCurrent` (0x191) reads
  −1 A (idle offset, no real flow), `BatVoltage` flat 359.75 V.
- t≈9.1: handshake group vanishes — IDs 040, 041, 117, 122, 3D8, 520, 67C
  present ONLY during this window. Candidate charge-session-coordination
  frames; re-check in a real DC session at low SoC before naming in DBC.
- t=22.5–22.8: 0x131 b3 = 0xFE sentinel burst (~15 frames) during/after the
  rejection. Confirms the firmware's sentinel guard (`SoC` comment in
  vwegolf.dbc / decode site) fires on real traffic — first FCAN observation.
- t≈104: drive-systems ID group stops (car settling back to idle).

### 0x569 on FCAN

Same ID as the KCAN OBC frame but near-static here (b1: 240→0 once at
t≈20, rest constant). No usable content at 100% SoC / no session.
Do NOT assume it mirrors the KCAN 0x569 OBC layout without a real session.

### Conclusion for charging-current hunt

Nothing decodable from this capture — no current flowed. Needs a real DC
fast-charge session at low SoC with filter off (short capture, supervised:
see storm caveats above).
