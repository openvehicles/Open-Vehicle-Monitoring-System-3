## Notes

DEFINITIVE hvac-source capture. Sequence: remote ON → run (thermostat cycles) → remote OFF.
(Live testing alongside this added the ignition/ACC observations below.)

**Signal semantics resolved — ms_v_env_hvac is driven by `0x3B5 ClimaRunning` (byte0 bit7).**

- `0x5EA ClimaRemoteMode` = "clima ECU **energized**", NOT conditioning. Live behavior:
  pinned true with ignition off after a remote start; flickers then holds true in radio/ACC
  mode; clears shortly after ignition off. So it couples to ignition — unusable for hvac.
- `0x3B5 ClimaRunning` (bit7) = blower actively conditioning, trigger-agnostic. High-rate
  (1262 frames). Tracks the real duty cycle: in this capture it cycled on/off **within** the
  session (on t=3.3, off 43.8, on 58.2, off 79.6, on 85.3, off 93.3) — thermostat behaviour.
- `0x66E ClimaRunning` (byte2 bit1, DBC's old "primary"): REJECTED — low-rate (~0.45 Hz),
  went off at t=43.8 and missed the later blower runs (58, 85). Unreliable.
- `0x17332510` port-0x18 ACK: transactional only (emits spurious `49 58 00`); OCU-ring use.

**Design shipped:** `0x3B5` drives ms_v_env_hvac with a **hold** (VWEGOLF_HVAC_RUN_HOLD_SECS,
20 s) to bridge thermostat cycling and clear on sleep; **command** sets the metric instantly
(responsive stop, no 20 s linger) and a **stop-suppress** window (VWEGOLF_HVAC_STOP_SUPPRESS_SECS,
10 s) ignores blower spin-down after a stop. Autonomous/timer stops linger ≈20 s (accepted).
